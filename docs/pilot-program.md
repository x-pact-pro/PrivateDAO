# Pilot Program

PrivateDAO is best introduced through a structured pilot, not an open-ended integration.

## Commercial offer

The judging-stage pilot offer is intentionally simple:

- price: `$2,500 setup`
- length: four weeks
- scope: one DAO or treasury workflow
- output: a working Testnet run, a proof packet, an operator handoff, and a conversion recommendation

This keeps the first paid motion concrete. A buyer does not need to buy a vague enterprise platform first; they buy one measurable privacy-sensitive operating workflow.

## Pilot profile

Ideal pilot organizations:

- DAOs with treasury approvals
- grant committees
- crypto-native funds
- gaming guilds
- enterprise-style internal operations teams

## Pilot structure

### Phase 1 — discovery and workflow selection

- identify one real governance flow
- choose one treasury action type
- decide whether confidentiality is required
- define the success metric and the stakeholder who must understand the proof

### Phase 2 — Testnet execution

- run the selected workflow on Solana Testnet
- capture evidence
- train the operator

### Phase 3 — proof packet and conversion

- define authority and signer model
- define monitoring expectations
- define launch blockers
- decide whether the buyer moves into managed operations, another pilot cycle, or stops before production expansion

## Pilot success criteria

- operator can complete DAO creation and treasury funding
- operator can submit and pass a proposal
- operator can execute treasury safely
- evidence packet is understandable to a non-builder stakeholder
- buyer can decide whether the workflow deserves a managed monthly plan

## Pilot exit conditions

- convert to managed operations starting at `$750/month`
- proceed to stronger security and launch readiness
- continue on Devnet for another cycle
- or stop before mainnet if the operating fit is not yet strong enough
